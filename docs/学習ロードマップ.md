## STEP1 Workspace + Compute 使い分け

見る場所：

* Workspace / Catalog / Compute / Workflows / SQL / ML

Compute 種類：

* Interactive compute（開発）
* Job compute（バッチ）
* SQL Warehouse（BI）

理解：

```text id="m2q5sa"
何でも1クラスターで済ませない
コスト事故はほぼ Compute 使い分けミス
```

---

## STEP2 Delta Lake（中核）

押さえる要素：

* ACID Transaction
* Schema Enforcement / Evolution
* Time Travel
* Merge / Update / Delete

理解：

```text id="x9p1rd"
S3置き場ではなく、Data Lake の自由さと DWH の整合性の中間
Databricks の中核はここ
```

---

## STEP3 Medallion 責務分離

```text id="v4r8tc"
Bronze raw
Silver cleaned
Gold business/ml ready
```

理解：

* 生データを壊さない
* 再処理性を確保する
* チーム責務を分ける

---

## STEP4 Workflows + Delta Live Tables

Workflows：

* Task / Dependency / Schedule / Retry / Parameter

DLT（Delta Live Tables）：

* 宣言的に Medallion を組む手段
* データ品質ルール（Expectations）

理解：

* Workflows は単なる cron ではなく責務連鎖の表現
* DLT は Bronze → Silver → Gold を宣言的に書ける

---

## STEP5 Unity Catalog

* Catalog / Schema / Table / View / Volume
* 権限 / Lineage / メタデータ統制
* モデル・特徴量テーブルまで包括

理解：

* Glue + IAM + Governance の進化版
* 企業導入時の決め手
