import json
import logging
import sys
import time
import uuid
from fastapi import FastAPI, Request

app = FastAPI()

logger = logging.getLogger("app")
logger.setLevel(logging.INFO)

handler = logging.StreamHandler(sys.stdout)
handler.setFormatter(logging.Formatter("%(message)s"))
logger.handlers = [handler]


def log_json(
    severity: str,
    message: str,
    **fields,
) -> None:
    payload = {
        "severity": severity,
        "message": message,
        **fields,
    }
    logger.info(json.dumps(payload, ensure_ascii=False))


@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    start = time.time()
    request_id = request.headers.get("x-request-id", str(uuid.uuid4()))

    try:
        response = await call_next(request)
        latency_ms = int((time.time() - start) * 1000)

        log_json(
            "INFO",
            "request completed",
            event="http_request",
            service="search-api",
            env="local",
            request_id=request_id,
            method=request.method,
            path=request.url.path,
            status_code=response.status_code,
            latency_ms=latency_ms,
            status="ok" if response.status_code < 500 else "error",
        )

        response.headers["x-request-id"] = request_id
        return response

    except Exception as e:
        latency_ms = int((time.time() - start) * 1000)

        log_json(
            "ERROR",
            "request failed",
            event="http_request",
            service="search-api",
            env="local",
            request_id=request_id,
            method=request.method,
            path=request.url.path,
            latency_ms=latency_ms,
            status="error",
            error_type=type(e).__name__,
            error_message=str(e),
        )
        raise

# ローカル Python ↔ Databricks Free Edition の接続

結論から言うと、**Workspace URL + Personal Access Token (PAT)** をローカルに登録しておけば、CLI も Python SDK も同じ認証情報を自動で拾います。3ステップです。

## 全体像

```
[ローカル WSL Python]
  ├─ databricks-cli  ──┐
  └─ databricks-sdk ───┤── HTTPS + PAT ──→ [Databricks Free Edition Workspace]
                       │                          ├─ Unity Catalog (Volumes, Tables)
   ~/.databrickscfg ───┘                          └─ Serverless Compute
   (host + token を保持)
```

ポイントは 「DatabricksのSDKは設定プロファイルや環境変数から認証情報を自動で拾うので、`WorkspaceClient()` を引数なしで呼べばよい」 という統一認証 (Unified Authentication) の仕組みです。CLI と SDK は同じ `~/.databrickscfg` を参照するため、一度設定すれば両方使えます。

## Step 1: PAT (Personal Access Token) を発行

Databricks Workspace の UI で：

1. 右上のユーザーアイコン → **Settings**
2. **Developer** → **Access tokens** の横の **Manage**
3. **Generate new token** → コメントと有効期限を入れて生成
4. 表示されたトークンをコピー（**この画面を閉じると二度と見られない**）

注意点：
- PAT は1ワークスペースあたり最大600個まで発行可能で、90日間未使用だと自動的に失効する
- Free Edition は1アカウント1ワークスペース、認証は email OTP / Google / Microsoft sign-in に限定、アカウントレベル API へのアクセス不可 — つまり `AccountClient` 系は使えず、`WorkspaceClient` のみ

## Step 2: ローカルで `databricks` CLI を入れて認証

```bash
# WSL での導入（公式インストーラ）
```

## Step 3: Python SDK から使う

```bash
# プロジェクトの venv 内で
pip install databricks-sdk
```

```python
# main.py
from databricks.sdk import WorkspaceClient

w = WorkspaceClient()  # ~/.databrickscfg の DEFAULT プロファイルを自動で拾う

# 動作確認
me = w.current_user.me()
print(f"authenticated as: {me.user_name}")

# Volume 操作
for item in w.files.list_directory_contents("/Volumes/workspace/default/raw_volume"):
    print(item.path)
```

`WorkspaceClient()` を引数なしで呼ぶだけで認証が通るのは、SDK が以下の優先順で認証情報を探してくれるからです：

1. 環境変数 (`DATABRICKS_HOST`, `DATABRICKS_TOKEN`)
2. `DATABRICKS_CONFIG_PROFILE` で指定されたプロファイル
3. `~/.databrickscfg` の DEFAULT プロファイル

SDK はクラウドプロバイダを自動検出して、必要な設定属性が揃っていない認証方式は自動でスキップするので、PAT さえ正しく設定されていれば他は気にしなくて大丈夫です。

## Doppler 派の流儀（推奨）

`~/.databrickscfg` にトークンを平文保存するのは Toshifumi さんの普段の運用と合わないので、Doppler に寄せる形：

```bash
# Doppler に登録
doppler secrets set DATABRICKS_HOST=https://dbc-xxxxxxxx-xxxx.cloud.databricks.com
doppler secrets set DATABRICKS_TOKEN=dapi************************

# 実行
doppler run -- python main.py
```

SDK 側は何も変えずに `WorkspaceClient()` のままで、環境変数経由で認証されます。CI/CD や Cloud Run へ持っていくときも同じパターンが使えるので、ローカルだけ `.databrickscfg`、本番は環境変数、みたいな分岐がなくなります。

## Free Edition で踏むハマりどころ

1. **Compute はサーバーレス専用** — カスタムcompute設定やGPUは非サポート、外向き通信も信頼ドメインに制限される。`spark.read.text(...)` 系は Notebook 内（サーバーレス）で動かす必要があり、ローカルから直接 Spark ジョブを叩くのは不可。
2. **ローカル Python の役割** — Volume へのファイル up/download、Job のキック、UC のメタデータ操作などの「コントロールプレーン」操作が中心。Spark 計算はワークスペース内で実行される。
3. **PAT の発行可否** — Free Edition でも UI から PAT は発行できるという報告が大半ですが、Free Edition の制約は変動が大きいため、もし Settings → Developer に Access tokens 項目が出てこなければ OAuth U2M (`databricks auth login --host <url>`) に切り替えれば同じことができます。

## 元の md について

貼ってもらった md にあった Volume アップロードのコードは、上記の Step 1〜3 が済んでいる前提で **そのまま動きます**。`WorkspaceClient()` の引数なし呼び出しが成立するための土台が今回の話、という関係です。

Phase 1 のローカル → Volume → Notebook 読み込みの流れは方向として妥当なので、まず `databricks current-user me` が通る状態を作るところがゴールになります。
